# GitHub App Onboarding And First Company Bootstrap

## Purpose

Define how a new operator connects GitHub and stands up the first company safely.

## Onboarding flow

1. explain what the GitHub App can read and write
2. install the app on chosen repositories
3. confirm installation and token health
4. create the first company
5. map repositories and initial agent set
6. validate projection and drift status

## First bootstrap checklist

- GitHub App installed
- company created
- repository mapping confirmed
- baseline policies selected
- projection health green
- first objective path understood

## UX rule

Onboarding must make permission scope, repository mapping, and first-company consequences explicit before work begins.
