# Base Operator UI

## Purpose

Define the first operator experience for understanding companies, agents, objectives, approvals, and current runtime state.

## Primary views

### Company overview

Shows:

- active objectives
- pending approvals
- recent failures
- projection health

### Objective workspace

Shows:

- work-item graph
- current owner
- blockers
- linked GitHub surfaces

### Run detail

Shows:

- execution packet summary
- artifacts
- validation result
- retry history

### Approval inbox

Shows:

- pending approvals
- requested action
- impact scope
- decision controls

## UI rule

The UI should explain what happened and what can happen next without requiring the operator to read hidden chat transcripts.
